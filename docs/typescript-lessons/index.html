Learn TypeScript by reading fjord

A progressive course that teaches TypeScript using real code from this codebase as the examples — so you learn the language and the project at the same time. Built for someone who knows Python well and TypeScript not at all.

Work top to bottom: each lesson assumes the previous ones. Code samples are quoted verbatim from the repo with a file:line caption so you can open the real thing alongside. Look for the coloured callouts:

  • 🐍 Python analogy — the nearest idea you already know.
  • ✓ Idiom & best practice — how good TypeScript is actually written.
  • ⚠ Gotcha — the traps that bite newcomers.
  • ⛓ In the fjord codebase — how the concept shows up here specifically.

The two lists this course braids together

The lessons deliberately interleave two syllabuses: the TypeScript concepts a good developer needs, taught through the implementation details you need to work on fjord.

1 · TypeScript concepts

basic → intermediate → advanced

  1. The type system, strict mode, structural typing, inference
  2. Interfaces & object shapes; optional vs nullable
  3. Union & literal types
  4. Functions, async/await, Promise<T>
  5. as const & deriving types from values
  6. Narrowing & type guards
  7. Generics & constraints
  8. Utility types (Pick, Omit, Partial…)
  9. Discriminated unions & exhaustiveness
  10. Classes, custom errors, unknown
  11. Modules, import type, the monorepo
  12. keyof/indexed access, Zod, Drizzle inference, satisfies
  13. React + TypeScript

2 · fjord implementation details

used as the examples, left to right

  • The shared workspace as the single source of truth
  • The five fixed Columns, derived from COLUMNS
  • The Task model & optimistic concurrency (version)
  • Domain validation via the Validated<T,C> result type
  • The blocking graph & derived blocked state
  • Config loading & validation with Zod
  • Drizzle schema & inferred row types
  • Typed domain errors → HTTP mapping
  • The Task-mutation seam (runTaskMutation)
  • The event bus & StreamEvent union; SSE filtering
  • The actor / auth model
  • The frontend api wrapper & ApiError
  • React Query, optimistic updates, context, useTaskEditor

The lessons

1 Why TypeScript exists Basic Static types over JavaScript, structural typing, inference, and the all-important fact that types vanish at runtime. 2 Interfaces & object shapes Basic Describing the shape of data with interfaces — optional fields, nullability, readonly, and extends. The Task and User models. 3 Union & literal types Basic The feature that makes TypeScript feel different from everything else: types built from specific values, combined with |. 4 Functions, async & Promises Basic Typing parameters and return values, default params, void/never, and the async/await + Promise<T> model that runs the backend. 5 as const & deriving types from values Intermediate The single-source-of-truth trick: write the list of values once, derive the type from it automatically. typeof and indexed access. 6 Narrowing & type guards Intermediate How TypeScript follows your control flow to turn 'string | null' into 'string'. Truthiness, typeof, instanceof, in, custom guards, and the ! escape hatch. 7 Generics Intermediate Functions and types parameterised by other types. The api.request<T> wrapper, the runTaskMutation<T> seam, and a generic <T> React component. 8 Utility types Intermediate Built-in type transformers — Pick, Omit, Partial, Readonly, Record — that build new types from existing ones instead of repeating yourself. 9 Discriminated unions & exhaustiveness Advanced The pattern that models 'one of several shapes, tagged by a field' — StreamEvent, the Validated result type — plus compiler-checked exhaustiveness with never. 10 Classes & error handling Intermediate Classes, parameter properties, and the custom-Error pattern that lets the service layer throw typed domain errors the routes map to HTTP — plus the transactional mutation seam. 11 Modules, imports & the monorepo Intermediate ES modules, named vs default exports, import type, the surprising .js extension on relative imports, and how @fjord/shared ties the workspaces together. 12 Advanced types at the runtime boundary Advanced keyof & indexed access, typeof-for-types, Zod runtime validation with inferred types, Drizzle's $inferSelect/$inferInsert, and the snake_case ↔ camelCase mapping seam. 13 React + TypeScript Advanced Where it all lands on the frontend: typed props, generic hooks (useState/useRef), typed events, React Query, typed context, and the useTaskEditor orchestrator.
